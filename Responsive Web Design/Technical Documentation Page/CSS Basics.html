Introduction

Cascading Style Sheets, commonly CSS, is the language used to style a web page. CSS is simple to understand, but also incredibly powerful in the way it can present an HTML document. In fact, this web page was written using primarily HTML and CSS.

The World Wide Web Consortium is the group of people who recommend how the internet works and evolves. A smaller group within known as the CSS Working Group create specifications for CSS.

The specification undergoes heavy discussion by the group members and eventually will become a recommendation, as mentioned above, if approved. These recommendations are also known as Web Standards. As a web developer, these standards serve as rules that allow everything to work together. Otherwise, you might end up needing a different browser every time you wanted to view a different web page!

Why use CSS?

CSS controls the style of a web page. A web page can exist without CSS, but it tends to look very basic and dull. With CSS, maintenance on a web page is fast and easy. It allows any part of the page to be styled, whether it's the layout, the color, the size, and much more.

CSS is created as it's own document so that the HTML can contain the content and isn't being over run with styling code.

This design allows a web designer to open up the stylesheet containing the CSS. They can clearly see each and every declaration block and edit the rule's defined within to make styling changes to the HTML sheet. Since the actual document isn't being altered in any way, it can be edited without fear of breaking the site's main functions.

The process is quick and efficient, and offers advanced tools that can be used as a framework for more specific and powerful languages too. Even though it would take a little more than an introduction page to master, CSS is easy enough to understand that many people can figure out how to do it on their own just by looking at the code!

Definitions

Info provided by impressivewebs.com

CSS has many terms that should be consistent among developers. Not every term applies only to CSS but when writing CSS code, it helps to be familiar with the following:

  • Comment

    • A block of text that is ignored by the browser. It's sole purpose is for the author to read or share messages.

  • Rule Set

    • A section of CSS code. It includes the selector and the declaration block. Example: 

    
          body {
          font-size: 10px;
          color: black
          }

  • Declaration Block

    • The text inside the {brackets} that tell the selector what to do.

  • Declaration

    • A single line of CSS, when grouped together they form a declaration block.

  • Property

    • The attribute of an element that CSS intends to style. It appears before the colon in a declaration.

  • Value

    • The styling to apply to a property. It appears after the colon.

  • Selector

    • The part of the CSS that selects what element is being targeted. This appears before the {brackets} and uses different symbols to determine what type of element it's looking for.

    • Element Selector

      • This refers directly to an element. It does not contain a symbol. Common element type selectors would include div, nav, a, body, for example.

    • Class Selector

      • This targets the element by the assigned class. It generally targets many elements of a related group. It is followed by a . symbol. Example: .navigation, .main-section, .container.

    • ID Selector

      • This targets an element by the id given. Unlike the class element selector it mainly targets a unique element. It is followed by the # symbol. Example: #slide001, #header-img, #jumbotron-content.

    • Universal Selector

      • Matches any element within the context it's placed in. An * is commonly used. Example: .container p * { width: 100px; } will make any element that descends from the container class's p element have a width of 100px. It can be used as a standalone selector to target the whole document.

    • Attribute Selector

      • This selects an element based on an attribute. Example: div[style] {background-color: red}; will target any div with the "style" attribute and make the background red. Another example: input[type="text"] {display: none;} will not display any input elements with a type attribute value of "text".

  • Pseudo-Class

    • This is similar to a regular CSS class but it is not declared in the HTML by the developer. It targets a specific element's state. It is always followed by a single colon. :visited, :active, :last-child, :in-range are all examples of a pseudo-class.

  • Pseudo-Element

    • A pseudo-element targets a specific part of an element. CSS2 pseudo-elements are followed by a single colon. CSS3 pseudo-elements are followed by a double colon. Example: span:first-line, p::selection.

  • Combinator

    • A symbol in a selector that connects two selectors together. The 4 symbols are >, +, ~, or a space.

  • At-Rule

    • An instruction given that uses the @ symbol. It could have a declaration block or a declaration. Example: @media screen and (min-width: 900px;).

  • Statement

    • Any at-rule or rule set.

  • Identifier

    • Anything that appears as a property, id, class, keyword value, and at-rule. Example: "body", "none", "font-size".

  • Keyword

    • A value for a property that can change. Example: auto, inherit.

Applying CSS

There are three methods at applying CSS to HTML. The most common method is to have a separate stylesheet containing your CSS and referencing it in the HTML document.

<link rel="stylesheet" href="styles.css">

This tells the HTML document that it needs to run the styles.css stylesheet. This will contain your CSS code and resemble this:

h1 {
color: orange;
background-image: linear-gradient(to-left, green, red);
font-size: 20px;
}
.container {
height: auto;
width: 40%;
}

How it Works

Info provided by developer.mozilla.org

So how does that one line of code tell HTML everything it needs to know to apply the CSS stylesheet? It all is quickly processed by the web browser in a rapid series of stages.

  1. The browser first loads the HTML
  2. The HTML is converted into a DOM (Document Object Model). The DOM is a tree structure that make up the parent and child nodes and their relationship to other DOM nodes.
  3. The browser fetches all of the external resources that are needed by the HTML document, such as images and videos, and linked CSS.
  4. The browser parses the CSS, sorting the rules by their selector types into "buckets". It then works out which rules should be applied to which nodes in the DOM and attaches the style to them. This step is called a render tree.
  5. The render tree is laid out in the determined structure after all the rules have been applied.
  6. The visual display is then shown on the screen in the painting stage.

Basic Commands

The syntax of CSS is as follows:

selector(s): {property: value; property: value; ...}

This is important to remember as all of your CSS coding is going to follow this format and the following commands will be presented this way.

A comment (remember, a note for people), is displayed as text surrounded by /* text */

Display

.element {
    display: none;            /* exiles the element from the page */
    display: block;           /* the default display style, can not be shared */
    display: inline;          /* allows the element to sit in a line */
    display: inline-block;    /* allows elements to sit together in line */
    }

Conclusion

As you can see, CSS is a powerful front-end styling tool. It cleanly manages data that's specifically used to present information from an HTML document. The tools offered by CSS are powerful, yet easy to use. Almost every web page you visit will have CSS somewhere and its a valuable language to master as a web developer. It is constantly evolving and has even acquired superset languages to further maximize it's potential. Effectively styling your document with CSS is only a beginning step in what the language can do, yet knowing how to code in CSS is a vital skill for any aspiring web developer to acquire.
